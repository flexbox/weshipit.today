## 5-Day Action Plan

---

### 🟦 Chunk 1 — Audit and Migrate to the New Architecture

**Goal:** Ensure the app runs fully on the New Architecture (no legacy bridge), making it future-proof and eligible for the latest RN 0.83+ features.

**Scope:**
- Audit all native modules and third-party libraries for New Architecture compatibility
- Enable `newArchEnabled: true` in `app.json` / `gradle.properties`
- Replace or patch any incompatible libraries using interop layer
- Run the app on both iOS and Android and verify no regressions

**Out of scope:** Migration of custom native modules requiring Swift/Kotlin rewrites.

**Dependencies:** React Native ≥ 0.83, Expo SDK ≥ 54 (or bare RN).

**Acceptance criteria:**
- App builds and runs on both platforms with `newArchEnabled: true`
- No bridge-related warnings in Metro/Hermes logs
- A non-technical stakeholder can open the app and navigate core flows without issues

**Estimated effort:** M

---

### 🟦 Chunk 2 — Configure rnrepo for Faster Android Builds

**Goal:** Reduce Android build times by up to 2× by integrating Software Mansion's rnrepo pre-built artifact registry.

**Scope:**
- Add rnrepo to the Gradle settings (`settings.gradle`)
- Configure the Maven repository URL pointing to rnrepo
- Verify cache hits on a fresh CI build
- Document the setup in the project README

**Out of scope:** iOS build optimisation (separate initiative), custom native module pre-building.

**Dependencies:** Android project, Gradle ≥ 7, existing CI pipeline (GitHub Actions or EAS Build).

**Acceptance criteria:**
- Cold Android build time drops by at least 40% compared to baseline
- CI logs show artifact download from rnrepo instead of local compilation
- A developer can reproduce the setup from the README in under 15 minutes

**Estimated effort:** S

---

### 🟧 Chunk 3 — Review Expo App Awards Winners and Apply UX Best Practices

**Goal:** Identify concrete UX and architecture patterns used by award-winning Expo apps and apply at least 2 improvements to the current app.

**Scope:**
- Browse the 2025 Expo App Awards winners (https://expo.dev/awards)
- Document 5+ UX or technical patterns observed across winners
- Implement 2 quick-win improvements (e.g., splash screen polish, native navigation feel, icon refinement)
- Write a brief internal review note shared in the team Notion

**Out of scope:** Full design system overhaul, new feature development.

**Dependencies:** Expo SDK ≥ 54, existing app shell.

**Acceptance criteria:**
- At least 2 UX improvements are live in the staging build
- Improvements are visible and verifiable by a non-technical stakeholder
- Internal review note is shared with the team

**Estimated effort:** S

---

### 🟦 Chunk 4 — Integrate react-native-tailwind for Zero-Runtime Styling

**Goal:** Replace or supplement the current styling approach with compile-time Tailwind CSS to eliminate runtime overhead and improve developer experience.

**Scope:**
- Install `react-native-tailwind` and configure the Babel/Metro plugin
- Migrate 2–3 existing screen components to Tailwind utility classes
- Verify no runtime style computation occurs (bundle analysis)
- Add a code-style guideline to the README

**Out of scope:** Full codebase migration, custom Tailwind plugin development.

**Dependencies:** React Native New Architecture enabled, Metro bundler.

**Acceptance criteria:**
- Target screens render correctly with Tailwind classes on both iOS and Android
- Bundle does not include runtime style evaluation for migrated components
- A developer can add a new Tailwind-styled component by following the README

**Estimated effort:** M

---

### ⚛️ Chunk 5 — Migrate Package Manager to Bun

**Goal:** Adopt Bun as the project's package manager and script runner to gain faster installs, built-in test runner support, and alignment with the Claude Code / AI-assisted dev workflow.

**Scope:**
- Replace `npm` / `yarn` / `pnpm` lock file with `bun.lockb`
- Update CI configuration to install dependencies via `bun install`
- Update project scripts in `package.json` to use `bun run`
- Verify all existing scripts (start, build, test, lint) work correctly

**Out of scope:** Migrating the Expo/Metro config to use Bun's bundler, replacing Node.js runtime entirely.

**Dependencies:** Bun ≥ 1.2, Node.js ≥ 20 still required for Expo/Metro.

**Acceptance criteria:**
- `bun install` completes successfully and reproduces the same dependency tree
- All CI checks pass using Bun
- Cold install time is measurably faster than the previous package manager
- A new developer can set up the project using only the README instructions

**Estimated effort:** S
